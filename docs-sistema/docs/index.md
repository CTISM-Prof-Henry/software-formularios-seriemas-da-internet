# Bem-vindo à Documentação

Este é o guia técnico oficial do **Sistema de Gerenciamento de Riscos e Unidades** da Universidade Federal de Santa Maria (UFSM).

---

## Propósito do Projeto
O sistema tem como objetivo principal digitalizar, mapear e gerenciar os riscos associados às mais de 5.000 unidades, departamentos e laboratórios da instituição. Ele centraliza as informações, permitindo a criação de formulários dinâmicos, relatórios de tratamento e auditoria.

## Perfis de Acesso
A plataforma é dividida em diferentes níveis de permissão:
* **Colaborador:** Responsável por preencher formulários e relatar dados básicos.
* **Gestor de Risco:** Analisa os dados do seu setor e elabora planos de tratamento.
* **Auditor:** Possui visão sistêmica para inspecionar, aprovar e auditar os processos de todas as unidades.

##  Tecnologias Utilizadas
A arquitetura do projeto é conteinerizada e dividida em microsserviços estruturais:
* **Backend:** Python, Django REST Framework
* **Banco de Dados:** PostgreSQL
* **Frontend:** React, React Router
* **Infraestrutura:** Docker e Docker Compose
